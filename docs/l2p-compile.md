# Compile — Learn to Play (Cue Card)

## Goal
- Be first to compile all 3 of your protocols.

## Setup
- Draft 3 protocols alternating (1+2, 2+1)
- Build deck: 6 command cards per protocol = 18 cards; shuffle.
- Place protocols “Loading…” side up in drafted order (3 lines total).
- Draw 5 cards each. First drafter takes first turn.

## Turn Order
- **Start:** Resolve your “Start” effects.
- **Check Control:** If you lead in total value in 2+ lines, take control.
- **Check Compile:** If eligible, you **must** compile (this is your action).
- **Action:** Play 1 card **or** Refresh (draw to 5).
- **Clear Cache:** If over 5 cards, discard down to 5.
- **End:** Resolve your “End” effects.

## Playing Cards
- Face-up: must match that line’s protocol; resolve immediate text.
- Face-down: can be played to any line.
- Top/bottom text works only while face-up (bottom usually needs uncovered).
- When active text enters play by being played, flipped face-up, or uncovered, 
resolve that text
- Targeting a **Covered Card** require text stating **Covered** or **All**


## Compile Rule
- During Check Compile: if your line has value **10+** and is higher than opponent’s same line, compile it.
- Both players delete all cards in that line to trash.
- Flip your protocol to “Compiled.”
- Only 1 compile per turn.
- Recompile an already-compiled protocol: clear line as normal, then draw top card of opponent’s deck (you own it).

## Core Reminders
- If you must draw and deck is empty: shuffle trash into new deck.
- Card text overrides base rules.
- Manipulation of **Covered Card** require text stating **Covered** or **All**

## Codex-Aligned Rules Nuances
- Refresh is only legal while you have fewer than 5 cards in hand.
- Auto-reshuffle is draw-only: if an effect needs the top deck card for a non-draw action and deck is empty, that effect step does nothing.
- Default targeting is uncovered-only unless text explicitly allows covered/all targets or points to an implied specific card.
- For "each" instructions, identify all valid objects first, then the resolving player picks the order and resolves one at a time.
- Start and End windows resolve as a queue: if an effect creates follow-up effects during that window, they continue resolving in the same window before leaving it.
- Draw/discard can be chained inside one resolution window; each queued step fully resolves before the next queued step.
